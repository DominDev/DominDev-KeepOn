# Code Signing Policy

KeepOn is an open source Windows desktop utility. This policy describes how
release artifacts are intended to be built, signed and published.

## Signing Provider

KeepOn is applying for free open source code signing through SignPath:

```text
Free code signing provided by SignPath.io, certificate by SignPath Foundation.
```

Until the SignPath Foundation application is accepted and the release workflow is
connected, published artifacts may be unsigned. Unsigned artifacts are marked as
such in the release notes or repository download section.

## Signing Scope

The intended signing scope is limited to official KeepOn release artifacts built
from this repository.

Files expected to be signed:

- `KeepOn.exe`
- KeepOn-owned binaries if additional project binaries are added in the future

Files not expected to be signed by this project:

- third-party runtime files,
- Microsoft .NET runtime files,
- source archives automatically generated by GitHub.

## Release Builds

Official release artifacts must be produced by GitHub Actions from the public
repository source code and release workflow.

The preferred corporate release artifact is:

```text
KeepOn-framework-dependent-win-x64.zip
```

The self-contained artifact is provided for machines without .NET already
installed:

```text
KeepOn-portable-self-contained-win-x64.zip
```

Release builds intentionally avoid compressed or packed single-file executables.
This makes the artifact easier for users and endpoint security products to
inspect.

## Approval Rules

Only tagged releases from the main repository should be submitted for production
code signing.

Before approving a signing request, the approver should confirm that:

- the release was built from the official repository,
- the version in `KeepOn.csproj` matches the release tag,
- release artifacts are not packed or compressed single-file executables,
- `SHA256SUMS.txt` is generated after final artifacts are created,
- the release does not add telemetry, hidden persistence, process injection,
  input simulation, downloaded payloads, services or drivers.

## Verification

Users and administrators can verify a signed release with:

```powershell
Get-AuthenticodeSignature .\KeepOn.exe
```

Expected result for signed releases:

```text
Status: Valid
SignerCertificate subject: SignPath Foundation
```

The exact certificate details may change according to SignPath Foundation
operations and certificate renewal.
